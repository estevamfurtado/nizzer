# nizzer

For now, I've been stuying web development for 1 month at [@Driven](https://www.driven.com.br/) (a 9-month fullstack web development bootcamp) where we covered a lot of html+css and part of js dynamics.

I decided to create a Trello clone with a few more features for educational purposes and personal use. In the absense of a better name, I'm calling it Nizzer (from organizer XD).

### A Trello ~kind of~ clone

Design, ux and major features/concepts (card, lists...) are inhereted from Trello. That said, the main difference is that all kanbans (as I call a list os lists) are going to be stacked in one single page - and the user will have the full ability to add, move and remove any element (cards, lists and kanbans).

*Checklists for the project:*
- [x] Have a similar feel to Trellos UI.
- [x] Set/get data in the localstorage.
- [x] Add and remove cards, lists and kanbans.
- [ ] Ability to move cards, lists and kanbans dragging and droping with a similar experience to Trello's.

### How I'm doing it

#### Structure

I've been learning at fast pace at Driven's bootcamp (check out my last project - [git](https://github.com/antcarlosrbj/projeto6-buzzquiz), [page](https://antcarlosrbj.github.io/projeto6-buzzquiz/)), but watching [this tutorial](https://www.youtube.com/watch?v=ijQ6dCughW8&t=2s) I realized that I'm not familiarized with these terms (classes, root, import), so I decided to incorporate that structure to build *nizzer*.

I also decided to use BEM as the class naming system for the first time in this project.

This combo is promoting forcing me to write a better code (much more clean).
